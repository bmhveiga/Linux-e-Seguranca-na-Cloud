# Tópico 05 - Atividade Prática Individual

## Tema

Monitorização, logs, backup, recuperação e continuidade operacional de um serviço web em Linux.

## Objetivo

Esta atividade tem como finalidade acompanhar o estado do sistema, consultar registos, criar backup e testar uma recuperação simples.

O foco foi:

- monitorizar recursos do sistema;
- verificar o estado do serviço web;
- consultar logs recentes;
- identificar eventos relevantes;
- criar backup;
- restaurar backup numa pasta de teste;
- definir um plano simples de continuidade operacional.

## Serviço analisado

O serviço crítico analisado foi o Nginx, usado para publicar um site HTML simples no servidor Ubuntu.

## Estrutura

- `monitorizacao.md`: verificação de uptime, memória, disco e serviço web.
- `logs.md`: consulta e interpretação de logs.
- `manutencao.md`: verificações básicas de manutenção.
- `backup-recuperacao.md`: criação e teste de recuperação de backup.
- `continuidade.md`: plano simples de continuidade operacional.
- `comandos.txt`: comandos usados.
- `evidencias/`: outputs seguros recolhidos.
